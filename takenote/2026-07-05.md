# Take Note - 2026-07-05

Hôm nay đã hoàn thành nâng cấp tính năng nạp Reference linh hoạt khi mở file, cấu trúc thư mục con Layout và tối ưu hóa giao diện danh sách file trong **Animeow Enjo Pipeline**:

## 1. Đa dạng hóa chế độ nạp Reference khi mở file
- Thay thế checkbox "Mở nhanh" cũ bằng dropdown **ComboBox** chứa 5 chế độ nạp Reference linh hoạt:
  1. `⚡ Mở nhanh (Không load Ref)`: Trì hoãn nạp tất cả reference để mở file nhanh nhất.
  2. `👤 Chọn nạp Nhân vật (Characters Only)`: Tự động lọc và tích chọn nạp rig nhân vật.
  3. `📋 Tự chọn Ref (Selective Preload)`: Hiển thị bảng chọn nạp gốc của Maya.
  4. `🔝 Chỉ nạp Ref cấp 1 (Top Level)`: Chỉ nạp reference ngoài cùng.
  5. `🔄 Mở bình thường (Nạp tất cả)`: Tải đầy đủ toàn bộ reference.
- **Lưu nhớ cấu hình (Persistence):** Tích hợp `cmds.optionVar` để ghi nhớ chế độ nạp được chọn cuối cùng của artist, tự động áp dụng khi khởi động lại công cụ.

## 2. Tích hợp bảng Preload tự động tích chọn Nhân vật/Rig
- Khi chọn chế độ **Chọn nạp Nhân vật**, file Maya sẽ được mở ở dạng trì hoãn nạp (`buildLoadSettings=True`).
- Hệ thống tự động phân tích và tích chọn nạp (`deferReference=0`) các file rig nhân vật/động vật trong cấu hình nạp ngầm, đồng thời bỏ chọn (`deferReference=1`) các rig bối cảnh/đạo cụ.
- Hiển thị bảng **Preload Reference Editor** của Maya chứa các thiết lập đã tích chọn sẵn để artist có quyền kiểm tra, tích chọn thêm bối cảnh hoặc bỏ bớt nhân vật trước khi tải scene chính thức.

## 3. Tối ưu bộ lọc nhận diện Rig nhân vật/động vật thông minh
- **Bộ lọc Loại trừ (Exclude) chạy trước:** Loại bỏ hoàn toàn các file/thư mục thuộc nhóm bối cảnh, đạo cụ (như chứa `/asset rig/`, `/bg/`, `/set/`, `/prop/`, `trung_tam_thuong_mai`...).
- **Bộ lọc Chấp nhận (Include) chạy sau:** Ưu tiên nhận diện các thư mục chứa rig nhân vật (`/character model/`, `/character rig/`, `/animal_rig/`...) hoặc tên file chứa từ khóa nhân vật (`baby`, `mom`, `dad`, `brother`, `sister`, `chicken`, `cow`...).
- Loại bỏ điều kiện dự phòng nạp chung chung theo từ khóa `rig` nằm trong thư mục `/05. RIG/` để khắc phục triệt để lỗi nạp nhầm các prop rig (như biển báo, cảnh báo, thang...).

## 4. Cấu trúc thư mục con theo tên shot cho khâu Layout
- **Quy hoạch ngăn nắp:** Riêng với khâu **Layout**, các file Maya nháp và file publish sạch được lưu trữ có hệ thống trong thư mục con đặt tên theo tên shot.
  - Ví dụ nháp: `WorkingFile/Layout/KS_ESS_V02_Shot_01-25/KS_ESS_V02_Shot_01-25_Lay_v02.ma`
  - Ví dụ publish: `Published/Layout/KS_ESS_V02_Shot_01-25/KS_ESS_V02_Shot_01-25_Lay_pub.ma`
- **Khâu Anim:** Giữ nguyên cấu trúc phẳng (lưu trực tiếp trong `WorkingFile/Anim/`) theo chuẩn làm việc tối giản.
- **Tương thích ngược:** Logic quét file được nâng cấp để hỗ trợ nạp cả các file cũ đang nằm ở ngoài thư mục cha của Layout.
- **Đồng bộ xuất Playblast:** Nâng cấp hàm `get_playblast_path` để tự động định vị đúng thư mục tập phim (`ep_dir`) dựa trên từ khóa `WorkingFile`, đảm bảo việc xuất video playblast nháp luôn chuẩn xác cho mọi khâu dù file nằm phẳng hay nằm sâu trong thư mục con.

## 5. Tự động kiểm tra và di chuyển file cũ vào folder con
- Tích hợp thêm tính năng rà soát vị trí file vào nút **Check Filenames (Kiểm tra tên file)** trên giao diện:
  - Tự động phát hiện các file Layout cũ đặt tên đúng quy chuẩn nhưng đang nằm trực tiếp ngoài thư mục cha.
  - Đề xuất di chuyển file đó vào đúng folder con chuẩn (tên file giữ nguyên, đường dẫn đích được tự động cập nhật).
  - Khi artist click xác nhận, tool tự động khởi tạo folder con (nếu chưa có) và di chuyển file vào đó một cách an toàn.

## 6. Thiết kế 2 bảng chọn Shot & Version tối ưu hóa giao diện
- Khắc phục tình trạng giao diện bị quá tải, rối mắt khi tập phim có hàng trăm shot và hàng nghìn file phiên bản cũ bằng thiết kế **Master-Detail (2 bảng song song)**:
  - **Cột bên trái (🎬 Danh sách Shot):** Chỉ hiển thị danh sách tên các Shot độc nhất trong khâu hiện tại.
  - **Cột bên phải (📂 Các phiên bản):** Khi chọn một Shot bên trái, cột bên phải hiển thị danh sách các file phiên bản tương ứng của shot đó (sắp xếp giảm dần theo version - phiên bản mới nhất hiển thị ngay trên cùng để đúp click mở).
- **Tự động đồng bộ và Highlight:** Khi mở file, giao diện tự động select Shot tương ứng ở cột trái và highlight file Maya đó ở cột phải.

## 7. Sửa lỗi tương thích kiểu dữ liệu trên Maya Python
- Khắc phục lỗi `'list' object has no attribute 'replace'` và `cannot concatenate 'str' and 'list'` phát sinh do hàm `cmds.selLoadSettings(..., fileName=True)` trong Maya Python trả về kết quả dưới dạng một `list` chứa 1 phần tử thay vì chuỗi.
