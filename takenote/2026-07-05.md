# Take Note - 2026-07-05

Hôm nay đã hoàn thành nâng cấp tính năng nạp Reference linh hoạt khi mở file trong **Animeow Enjo Pipeline**, giúp tối ưu hoá hiệu năng làm việc và tăng tính kiểm soát cho artist:

## 1. Đa dạng hóa chế độ nạp Reference khi mở file
- Thay thế checkbox "Mở nhanh" cũ bằng dropdown **ComboBox** chứa 5 chế độ nạp Reference linh hoạt:
  1. `⚡ Mở nhanh (Không load Ref)`: Trì hoãn nạp tất cả reference để mở file nhanh nhất.
  2. `👤 Chọn nạp Nhân vật (Characters Only)`: Tự động lọc và tích chọn nạp rig nhân vật.
  3. `📋 Tự chọn Ref (Selective Preload)`: Hiển thị bảng chọn nạp gốc của Maya.
  4. `🔝 Chỉ nạp Ref cấp 1 (Top Level)`: Chỉ nạp reference ngoài cùng.
  5. `🔄 Mở bình thường (Nạp tất cả)`: Tải đầy đủ toàn bộ reference.
- **Lưu nhớ cấu hình (Persistence):** Tích hợp `cmds.optionVar` để ghi nhớ chế độ nạp được chọn cuối cùng của artist, tự động áp dụng khi khởi động lại công cụ.

## 2. Tích hợp bảng Preload tự động tích chọn Nhân vật/Rig
- Khi chọn chế độ **Chọn nạp Nhân vật**, file Maya sẽ được mở ở dạng trì hoãn nạp (`buildLoadSettings=True`).
- Hệ thống tự động phân tích và tích chọn nạp (`deferReference=0`) các file rig nhân vật/động vật trong cấu hình nạp ngầm, đồng thời bỏ chọn (`deferReference=1`) các rig bối cảnh/đạo cụ.
- Hiển thị bảng **Preload Reference Editor** của Maya chứa các thiết lập đã tích chọn sẵn để artist có quyền kiểm tra, tích chọn thêm bối cảnh hoặc bỏ bớt nhân vật trước khi tải scene chính thức.

## 3. Tối ưu bộ lọc nhận diện Rig nhân vật/động vật thông minh
- **Bộ lọc Loại trừ (Exclude) chạy trước:** Loại bỏ hoàn toàn các file/thư mục thuộc nhóm bối cảnh, đạo cụ (như chứa `/asset rig/`, `/bg/`, `/set/`, `/prop/`, `trung_tam_thuong_mai`...).
- **Bộ lọc Chấp nhận (Include) chạy sau:** Ưu tiên nhận diện các thư mục chứa rig nhân vật (`/character model/`, `/character rig/`, `/animal_rig/`...) hoặc tên file chứa từ khóa nhân vật (`baby`, `mom`, `dad`, `brother`, `sister`, `chicken`, `cow`...).
- Loại bỏ điều kiện dự phòng nạp chung chung theo từ khóa `rig` nằm trong thư mục `/05. RIG/` để khắc phục triệt để lỗi nạp nhầm các prop rig (như biển báo, cảnh báo, thang...).

## 4. Sửa lỗi tương thích kiểu dữ liệu trên Maya Python
- Khắc phục lỗi `'list' object has no attribute 'replace'` và `cannot concatenate 'str' and 'list'` phát sinh trong quá trình chạy thực tế.
- Nguyên nhân do hàm `cmds.selLoadSettings(..., fileName=True)` trong Maya Python trả về kết quả dưới dạng một `list` chứa 1 phần tử thay vì chuỗi. Đã được xử lý triệt để bằng cách tự động unwrap phần tử đầu tiên của list thành chuỗi trước khi truyền vào các hàm xử lý logic đường dẫn.
